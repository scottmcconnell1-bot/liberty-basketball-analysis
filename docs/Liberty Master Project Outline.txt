------------------------------

## 1.1 Existing Analysis Tables (do not break)

These tables already exist and support the current tagging/analysis flow:

- events: in-game events (possessions, dribbles, shots).  
- analysis_runs: status of video analysis runs.

Hermes must not drop or fundamentally change these without an explicit plan. [ppl-ai-file-upload.s3.amazonaws](https://ppl-ai-file-upload.s3.amazonaws.com/web/direct-files/attachments/168271520/22ac67b5-62fd-45e2-a45c-fdbb641f93d2/Liberty-Master-Project-Outline-1.txt)

## 1.2 Seasons, Schedules, Games, and Sources (DONE)

Goal: Represent my world of seasons, games, NFHS/Pixellot/manual film, and multiple sources per game in a clean schema. [ppl-ai-file-upload.s3.amazonaws](https://ppl-ai-file-upload.s3.amazonaws.com/web/direct-files/attachments/168271520/ac0b75d5-c626-454c-9c35-b949fc19ce35/Liberty-Master-Project-Outline.txt)

Tables:

- seasons  
  - id (PK)  
  - name (TEXT, UNIQUE) -- for example, 2025-26.  
  - start_date (DATE)  
  - end_date (DATE)  
  - created_at (TIMESTAMP)

- scheduled_games  
  - id (PK)  
  - season_id → seasons.id (NOT NULL)  
  - program_name (TEXT) -- for example, Liberty.  
  - gender (TEXT) -- boys or girls.  
  - level (TEXT) -- varsity, jv, jr_high.  
  - game_date (DATE)  
  - game_time (TIME)  
  - location_type (TEXT) -- home, away, neutral.  
  - opponent_name (TEXT)  
  - tournament_name (TEXT, optional)  
  - status (TEXT, default scheduled) -- scheduled, cancelled, rescheduled, completed.  
  - notes (TEXT)  
  - created_at, updated_at (TIMESTAMP)

- games  
  - id (PK)  
  - scheduled_game_id → scheduled_games.id (nullable, supports scouting games not on my schedule).  
  - start_time (TIMESTAMP, optional)  
  - end_time (TIMESTAMP, optional)  
  - source_type (TEXT, NOT NULL) -- pixellot, nfhs, manual.  
  - source_key (TEXT, NOT NULL) -- NFHS game_id, Pixellot id, or local path.  
  - nfhs_game_id (TEXT, optional)  
  - nfhs_url (TEXT, optional)  
  - created_at, updated_at (TIMESTAMP)

- nfhs_matches  
  - id (PK)  
  - scheduled_game_id → scheduled_games.id (NOT NULL)  
  - nfhs_game_id (TEXT, NOT NULL)  
  - nfhs_url (TEXT, NOT NULL)  
  - match_status (TEXT, default candidate) -- candidate, confirmed, rejected.  
  - confidence (REAL, optional)  
  - created_at (TIMESTAMP)

- sources  
  - id (PK)  
  - game_id → games.id (NOT NULL)  
  - source_type (TEXT, NOT NULL) -- nfhs_vod, manual_upload, local_file, etc.  
  - source_path (TEXT, NOT NULL) -- file path or URL.  
  - created_at, updated_at (TIMESTAMP)

Status: Implemented in schema.sql with CREATE TABLE IF NOT EXISTS. This structure is locked in as the basis for all schedule, game, and film features. [ppl-ai-file-upload.s3.amazonaws](https://ppl-ai-file-upload.s3.amazonaws.com/web/direct-files/attachments/168271520/22ac67b5-62fd-45e2-a45c-fdbb641f93d2/Liberty-Master-Project-Outline-1.txt)
