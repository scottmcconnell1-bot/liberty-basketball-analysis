  - Have a fallback if automation fails (I can still watch film and take notes).
- If a feature is not used in **10 games or 10 practices**, we either remove it or rethink it.

### 0.4 Feature Flags & Compartmentalization

Add a shared configuration pattern (example; actual code in `config.py`):

```python
class Features:
    ENABLE_MANUAL_TAG_MVP = True
    ENABLE_AUTO_STATS_M1 = True
    ENABLE_EXTENDED_EVENTS_M2 = False
    ENABLE_WEEKLY_PACKET = False
    ENABLE_SEASON_REVIEW = False

class AnalysisConfig:
    USE_DRIBBLE_EVENTS = False      # default: no dribble events as stats
    USE_DRIBBLE_HEURISTICS = True   # internal only, if needed
```

Rules:

- Every major feature/phase is behind a **single flag** so it can be disabled with minimal impact.
- Optional features get their own routes/templates/modules so they can be removed or refactored safely.
